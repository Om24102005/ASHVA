# ASHVA — Deploy backend to Render

Render deploys from a **GitHub repo**. Flow: push repo → Render Blueprint reads
`render.yaml` → provisions Postgres + the API → you set secrets in the dashboard
→ point the app at the Render URL.

## 0. Prereqs (one-time)
- A **Render account** (the email you'll connect — tell me which).
- A **GitHub repo** for this project (I'll init git + push; `.env` is now gitignored).
- **Cloudflare R2** bucket for KYC docs (S3-compatible; Render has no managed S3).

## 1. Push to GitHub  *(I do this once you OK it)*
Secrets are gitignored now. Before pushing I will **rotate the leaked creds**
(DB password, MSG91 authkey, MinIO/R2 keys) — they sat in plaintext. `JWT_SECRET`
is auto-regenerated by Render (see render.yaml `generateValue`).

```bash
git init && git add -A && git commit -m "ASHVA: API + RN app"
git remote add origin git@github.com:<you>/ashva.git
git push -u origin main
```

## 2. Render Blueprint
Dashboard → **New → Blueprint** → connect the repo → **Apply**.
This creates `ashva-db` (Postgres) + `ashva-api` (Docker web service) and wires
`PG*` env vars automatically.

## 3. Set secrets in the Render dashboard  (ashva-api → Environment)
These are `sync: false` in render.yaml (never in git):
- `MSG91_AUTHKEY`, `MSG91_SMS_TEMPLATE_ID`, `MSG91_EMAIL_FROM`, `MSG91_EMAIL_DOMAIN`, `MSG91_EMAIL_TEMPLATE_ID`
- `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`  (Gmail app password → reliable email OTP)
- `GOOGLE_CLIENT_IDS`  (web,iOS[,Android])
- `S3_ENDPOINT` = `https://<acct>.r2.cloudflarestorage.com`
- `S3_PUBLIC_URL` = your R2 public bucket URL (https)
- `S3_ACCESS_KEY`, `S3_SECRET_KEY`  (R2 API token)

## 4. Initialize the database schema (once)
Render managed Postgres does NOT auto-run `server/init.sql`. Run it once:
```bash
# copy the External Database URL from the ashva-db page
psql "<EXTERNAL_DATABASE_URL>" -f server/init.sql
```

## 5. Point the app at Render
In `ashva-rn/.env`:
```
EXPO_PUBLIC_API_URL=https://ashva-api.onrender.com
```
Rebuild the app. It now works on **any** network — no IP, ever.

## Notes
- Free Postgres expires in 30 days; use `starter` for anything real.
- Free web services cold-start (~30s after idle); first request may be slow.
- KYC bucket: keep it **private** + serve via signed URLs (the audit's Critical #2).
  Public-read R2 repeats the same PII leak.
